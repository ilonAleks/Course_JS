Task 14-5

In order to successfully complete all your tasks on the project, it remains only to complete the deletion of tasks. To do this, there is a “delete” button. But sometimes users can accidentally press the button and the task will be removed because of this. For this, modal windows are usually created that ask the user, whether he really wants to remove the task. Now you will need to implement such a modal window.

You already have styles for the modal window and the HTML step. Create the HTML-element of the modal window using JavaScript and add it inside the tag body.

Modal window template:

<div class="modal-overlay modal-overlay_hidden"> <div class="delete-modal"> <h3 class="delete-modal__question"> Do you really want to delete this task? </h3> <div class="delete-modal__buttons"> <button class="delete-modal__button delete-modal__cancel-button"> Cancel </button> <button class="delete-modal__button delete-modal__confirm-button"> Delete </button> </div> </div> </div> Class “modal-overlay_hidden” is responsible for the opening and closing of the modal window. If it is open, then the class “modal-overlay_hidden” is not, if closed, then the class “modal-overlay_hidden” is present.

You need to open a modal window after clicking on any button “Delete” near each task. Hang the processor of the event “click” on the element with the class “Tasks-list” . Use delegation of events to catch clicks on buttons to delete tasks.

There are 2 buttons in the modal window: “cancel” and “delete”. “Cancel” closes the modal window, and “delete” - removes the task from the TASKS array and from DOM -Derev and closes the modal window.

To delete the task use the attribute data-task-id, which contains information about the property id object of the task from array tasks. According to this id you can find the desired task and delete it.
