Task 19-5

Congratulations! You have almost completely implemented the donation application. You now have the logic of creating donations and rendering all the necessary HTML elements.

Now it remains to work a little with the quality of the written code.

To begin with, let's note that we have all the amounts written in dollars "$". And imagine that a customer comes and says: "Let's all the amounts be in euros." With the current implementation, you will have to fix the dollar sign to the euro in all possible files, which is of course inconvenient. There is a solution for this. Create a global value and use it throughout the project.

Therefore, in the core folder, create a constants folder. In the constants folder, create a settings.js file that will store the Settings object. In this object, create a currency key, which will have the value “$”. Export Settings and then use import to replace all "$" signs in the files with Settings.currency.

After the work done, you can safely change the value of the currency property in the Settings object, and the currency will change throughout the project.

Condition: when importing, you must use the as keyword at least once. For example, import { Object as Obj }.
