Task 13-6

You are well done! You have implemented logic for two dictionaries and they work perfectly. But then a senior developer comes to you and says that you need to use encapsulation in your code, as this is an important principle of OOP.

You certainly agree with the senior developer. He advised you to make the fields name and words private. Implement this using the sign of the lattice “#”.

Now you need to add getters and setters for fields words and name in class Dictionary. For name, name the getter mainName, and setter - setMainName. For words name the getter allWords, and setter - addNewWord. mainName, setMainName and allWords should be defined using keywords set or get.

But addNewWord is necessary to create as a regular method that accepts 2 parameters: word - word, type of data string; description - word description, data type string. Method addNewWord should create a new word in the object words without any checks. addNewWord realized without set, because set cannot take more than the 1st parameter.

After you made the fields private and created getters and setters, you need to fix logic for classes Dictionary and HardWordsDictionary.

Test the final code on this example:

const hardWordsDictionary = new HardWordsDictionary('Difficult words');
hardWordsDictionary.add('Amateur', 'anyone who is engaged in science or art without special training, possessing only superficial knowledge.');
hardWordsDictionary.add('Neologism', 'new word or expression, as well as the new meaning of the old word.');
hardWordsDictionary.add('Quantum', 'the indivisible part of any value in physics.');

hardWordsDictionary.remove('Neologism');
hardWordsDictionary.showAllWords();

console.log(hardWordsDictionary.mainName); // Difficult words
hardWordsDictionary.setMainName = 'New Dictionary';
console.log(hardWordsDictionary.mainName); // New Dictionary
console.log(hardWordsDictionary.allWords); //displays an object in which there are words Amateur and Quantum
