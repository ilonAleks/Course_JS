Task 12-Additional

One owner of a small startup came to you. He offers you to make a game that will amaze the game industry. You certainly agree, since the proposal sounds tempting.

The essence of the game is that there are 2 castles (attacking and defending) that are at war together. User play for the attacking castle. The king of the attacking castle must every time calculate the possibilities of capturing the warring castle. When the possibility is large enough, you must not attack without hesitation.

In the code you have 2 objects initially attacker and defender which are the analogies of locks and which have properties responsible for the number of combat units of each type. For example, archer - these are archers, footSoldiers - these are infantrymens, etc.

const attacker = {
  archer: 30,
  footSoldier: 55,
  cavalry: 10,
  artillery: 3,
}

const defender = {
  archer: 33,
  footSoldier: 50,
  cavalry: 40,
  artillery: 10,
}

In the Attacker object, you need to create 3 functions:

  1. checkChancesToWin. Checks the chances of an attacking castle to capture a defending castle.This function accepts 1 parameter defenderObject, which is an object defender.
    checkChancesToWin The number of combat units from both castles must be verified.For example, if the attacking castle has a greater value of the Archer property than that of the defender, then 1 must add 1 to the chances of capture (initially the value of chances should be zero).
    The function should return an array in which the 1st element is the chances of an attacking castle for capture, and the 2nd is the value of the maximum chance of capture (the number of keys in the defenderObject object).That is, if the attacking castle of the properties of Archer and Cavalry has more than the defending, and the other values of the properties are less, then the chances of capture should be [2, 4] (figuratively the chances are 2 of 4 or 50%).
  2. improveArmy. Adds to each numerical value of the Attacker object by 5 units (updates the properties of Archer, FootSoldier, Cavalry, Artillery). This function does not accept any parameters.
  3. attack. It takes 1 parameter - this is the Defender object. First, this function checks if the chances of capture (values from the function checkChancesToWin) less than 70% of the maximum chances, then it is necessary to intensify (call the function improveArmy) and display a message via Alert “Our chances are equal ${ourArmyChances}/${maximumChances}. Must be strengthened!” (ourArmyChances - the chances of an attacking castle for capture, maximumChances - the maximum chance of capture). Otherwise, you need to display a message in the modal window “We intensified! We will undoubtedly win! Our chances are high!”.
Example result function of function attack:
attacker.attack(defender); // Our chances are equal 1/4. Must be strengthened!
attacker.attack(defender); // Our chances are equal 2/4. Must be strengthened!
attacker.attack(defender); // We intensified! We will undoubtedly win! Our chances are high!
